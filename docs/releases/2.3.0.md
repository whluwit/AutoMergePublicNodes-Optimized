# AutoMergePublicNodes-Optimized 2.3.0 发布说明

## 概要

2.3.0 将项目从"按延迟排序的节点输出流水线"升级为"可配置、可观测的节点质量流水线"。默认行为保持兼容：GitHub Actions 仍使用 `config/scoring.yaml`，真测仍决定节点是否可用，评分仅控制可用节点的排序。

## 主要变化

### 可配置评分

节点排序不再只看延迟，而是使用加权评分配置。默认配置：

```text
config/scoring.yaml
```

默认权重：

| 因子 | 权重 | 含义 |
|---|---:|---|
| `latency` | 35 | 真实代理测试延迟 |
| `jitter` | 15 | 多目标真测抖动 |
| `tcp` | 10 | TCP 预筛选延迟 |
| `protocol_history` | 20 | 协议历史通过率 |
| `source_history` | 20 | 订阅源历史通过率 |

### 评分模板

针对不同场景提供额外模板：

| 模板 | 适用场景 |
|---|---|
| `config/scoring.low_latency.yaml` | 优先选择最快节点 |
| `config/scoring.stability.yaml` | 优先选择低抖动稳定节点 |
| `config/scoring.source_quality.yaml` | 优先选择可信来源和协议 |

本地运行指定模板：

```bash
python main.py --top-n 100 --test-limit 500 --scoring-rules config/scoring.stability.yaml
```

### 可观测性增强

新增或增强的输出：

| 输出 | 用途 |
|---|---|
| `output/stats.json` | 包含评分配置和 Top 节点分数拆解 |
| `output/health_report.md` | 面向人工阅读的健康报告，含评分详情 |
| `output/scoring_profiles.md` | 评分模板对比表 |
| GitHub Actions Summary | 运行概览、评分权重、分数拆解、输出链接 |
| README 自动状态块 | 仓库首页直接展示最新运行状态 |

### CI 与校验

- CI 会校验全部 `config/scoring*.yaml` 模板。
- 权重总和不等于 100 时只给 warning，不阻断 CI。
- 回归测试覆盖评分配置、分项账目、健康报告、评分模板报告、Actions Summary。

## 兼容性

无需更改订阅 URL，现有用户可继续使用：

```text
output/verified.txt
output/verified.yaml
output/verified.json
```

唯一行为变化是排序：可用节点先按综合质量得分排序，延迟作为同分时的决胜项。

## 推荐验证

```bash
python -m compileall -q main.py core tools tests
python tools/validate_config.py --sources config/sources.yaml --filter-rules config/filter_rules.yaml --scoring-rules 'config/scoring*.yaml'
python -m unittest -v tests.test_regressions
python tools/scoring_profiles_report.py --profiles 'config/scoring*.yaml' --output output/scoring_profiles.md
```
