# AutoMergePublicNodes-Optimized 2.4.0 发布说明

## 概述

2.4.0 在 2.3.0 的评分体系和可观测性基础上，全面补齐了输出分发、容错降级、测试增强和社区入口。本项目现在从「可配置、可观测的节点质量流水线」升级为「可分发、可容错、可持续运营的节点质量平台」。

默认行为完全兼容：GitHub Actions 仍使用 `config/scoring.yaml`，真实代理测试仍决定节点是否可用，评分仅排序可用节点。

## 新增功能

### 分块输出

verified 和 global 节点现在按每 100 个一组拆分为独立订阅文件：

```
output/chunks/verified_1.txt   # 第 1-100 个节点
output/chunks/verified_2.txt   # 第 101-200 个节点
output/chunks/verified_3.txt   # 第 201-300 个节点
```

每个文件是独立的 base64 订阅，可单独导入客户端。避免单文件过大导致 CDN 缓存延迟。

### 协议分文件输出

verified 和 global 节点现在按协议类型分文件输出：

```
output/by_protocol/verified_vmess.txt
output/by_protocol/verified_vless.txt
output/by_protocol/verified_trojan.txt
output/by_protocol/verified_shadowsocks.txt
```

方便只需要特定协议节点的用户。

### 节点稳定性追踪

新增 `output/node_stability.json`，记录每个节点指纹的跨轮连续通过/失败历史。连续通过多轮的稳定节点在评分时获得优先标记。最多追踪 5000 个节点指纹，超出后按最近活跃时间淘汰。

### Netflix 和 ChatGPT 解锁检测

真实测试通过后，额外检测节点是否能访问 Netflix 和 ChatGPT。结果记录在 `stats.json` 的 `top_scores` 中（`netflix` 和 `chatgpt` 字段），不影响节点是否通过测试。

### 下载速度量化

`stats.json` 的 Top 节点新增 `speed_kbps` 字段，记录 100KB 下载测试的实际速度。

### 延迟测量精度提升

对 YouTube 和 Google 的 204 检测目标从单次请求改为三次采样取中位数，减少网络抖动导致的延迟偏差。

### 源自动发现

新增 `tools/source_discovery.py`，扫描 6 个高 star 公开节点项目的订阅链接，统计发现的候选节点数量和协议分布，输出到 `output/source_discovery.json`。不自动加入 `sources.yaml`，供维护者参考。

### CI 失败通知和恢复

- CI 失败时自动创建 GitHub Issue（标签 `ci-failure`），包含运行链接和触发信息。
- CI 恢复成功时自动关闭所有 `ci-failure` Issue 并评论「✅ CI recovered」。

### 优雅降级

真测 0 通过时不再输出空订阅，而是自动降级到 TCP 可达节点填充输出，`stats.json` 标记 `degraded_mode: true`，确保用户始终有节点可用。

### sing-box 下载容错

sing-box 最新版下载失败时，自动尝试 3 个旧版本（v1.11.15、v1.10.7、v1.9.7），避免因单次下载失败导致整个流水线崩溃。

### 社区贡献源入口

新增 GitHub Issue 模板 `.github/ISSUE_TEMPLATE/new-source.yml`，用户可以通过 Issue 推荐新订阅源，填写 URL、名称、格式和预估节点数。

### Cloudflare Worker 加速方案

新增 `docs/cloudflare-worker-setup.md`，提供完整的 Cloudflare Worker 脚本和部署指南，作为 jsDelivr CDN 的补充加速方案。

## 兼容性

无需更改任何订阅地址。现有用户可以继续使用：

```
output/verified.txt
output/verified.yaml
output/verified.json
```

新增的分块和协议分文件是额外输出，不影响原有订阅。

## 推荐验证

```bash
python -m compileall -q main.py core tools tests
python tools/validate_config.py --sources config/sources.yaml --filter-rules config/filter_rules.yaml --scoring-rules 'config/scoring*.yaml'
python -m unittest -v tests.test_regressions
python tools/scoring_profiles_report.py --profiles 'config/scoring*.yaml' --output output/scoring_profiles.md
```

## 注意事项

- README 运行时状态由定时 GitHub Actions 运行生成，显示的版本和数量可能在下一次运行后才更新。
- 分块和协议分文件在 CI 运行时自动生成，首次推送后通过 jsDelivr 可访问。
- Netflix 和 ChatGPT 解锁检测结果仅供参考，不代表节点在所有网络环境下都能解锁。
